Creating the application

A CAF application is created by calling the wd.caf.application constructor. The specs for a new application include the name, the html container where the application will run, the template and the type of transition between panels. 

var application = wd.caf.application({
    name: 'Community Application Framework', 
    container:"#caf", 
    template:'default', 
    transition: "basic"
});

Elements

All CAF elements extend the same base class and share the same syntax for creation. TODO:

Luckily, CAF already implements default classes for most of the UI elements and a CAF application comes with those default elements registered. The exceptions are panels and actions, since they directly affect the layout. So, to create a new CAF application the first step is to register panels and actions.

Panels

This sample panel is built using the pre-defined externalDivContent panel class, which takes a DOM element from a given url and uses it as content for the registered panels. 

application.getRegistry().registerPanel( wd.caf.impl.panels.externalDivContent({
    name:'about',
    description: 'About',
    order:"20", 
    url:"panelAbout.html", 
    selector:"#about"
}) );

Actions

This sample action is registered using the same syntax as the panels. The executeAction function triggers one of the registered notification messages. 

application.getRegistry().registerAction(wd.caf.action({
    name: "notificationAction",
    description: "Notification action",
    order: 105,
    executeAction: function(){
        this.caf.notificationEngine.getNotification().success("Hello world")
    }
}) );

Keybinds [keybinds are optional but share the same syntax]

These sample keybinds use a pre-defined panelKeybind class, which triggers a transition to a given panel

application.getRegistry().registerKeybind(  wd.caf.impl.keybinds.panelKeybind({
    name:'about',
    key:'1'
}) );
