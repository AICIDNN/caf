Create a new action

Just like panels, building a custom action class is done by extending another action class, particularly wf.caf.action .

new specs
Redefinition of the specs of the parent action class and addition of new ones.
parent class
The action class to extend.
executeAction( )
The function that implements the action itself.

Two ways to do it

Complete syntax [create a new action constructor]

wd.caf.impl.actions.sampleAction = function(spec){
    
    /**
     * Specific specs
     */
    
    var _spec = {
        name: "sampleAction",
        description: "Sample alert action",
        order: 100
    };

    spec = $.extend({},_spec,spec);  
    var myself = wd.caf.action(spec);
   
    myself.executeAction = function(){
        
        alert("Hello world");
    }
        
    return myself;
        
};

Register the action 

wd.caf.registry.registerAction(wd.caf.impl.actions.sampleAction() );

Simpler syntax [pass options to an existing action constructor and register directly]

wd.caf.registry.registerAction( wd.caf.action({
    name: "sampleAction",
    description: "Sample notification action",
    order: 105,
    executeAction: function(){
        this.caf.notificationEngine.getNotification().success("Hello world")
    }
}) );
