Elements

CAF implements a set of different UI elements:

Panels
Panels are the placeholders for the application content. Navigation between panels is, by default, implemented in the top right corner. CAF includes several panel classes that can get the content directly from the panel definition, from a html element, from an html element in another file or using an iframe.
Actions
Actions are user triggered operations which, by default, are available on the top left corner of the UI. Examples of actions are things like Save, Open, Log out...
Notifications
These are messages that can be triggered by some another element. CAF implements a default notification type with success, info, warn, debug and error notification levels.
Popups
The general popup interface implements functions to show, hide and draw a popup. CAF includes more complex implementations like ok/cancel popups or close popups.
Templates
CAF templates define the general application layout, particularly where to draw actions and panels.
Keybinds
With CAF keybinds, a combination of keys can be bound to a given operation. Particularly, they can be used to define a quick way of switching between panels.
